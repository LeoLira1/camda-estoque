# Product

## Register

product

## Users

Gestor único da cooperativa CAMDA (Quirinópolis, GO), revisando dados operacionais no navegador desktop. Contexto de uso: mesa de trabalho, janela do dashboard aberta durante o expediente, tomada de decisões sobre estoque real de defensivos, sementes e insumos agrícolas. Não é uso casual — é ferramenta de trabalho diária, com dados que têm consequência direta na operação do armazém.

## Product Purpose

Dashboard interno de gestão de estoque para a cooperativa CAMDA. Centraliza: visão geral do inventário por status (OK/divergência/pendente), inventário cíclico com conferência produto a produto, mapa físico do armazém, mural de avisos da equipe, alertas de validade e reposição, e carrossel de informações do agronegócio (clima, preços CEPEA, notícias). Sucesso é o gestor sabendo em segundos o estado real do armazém — sem precisar checar planilhas ou ligar para o depósito.

## Brand Personality

Instrumento preciso, enraizado, confiável.

## Anti-references

- **Grafana / Datadog**: azul-marinho monótono, grade de cards idênticos, sensação de template genérico sem identidade. Parece que poderia ser qualquer produto para qualquer setor.
- **PowerBI / Tableau**: caos visual com gráficos em arco-íris, dashboards que gritam em vez de comunicar.
- **Notion / Linear minimalistas**: espaço em branco excessivo inadequado para dados densos e operacionais. Parece inacabado para uma ferramenta de armazém.
- **"Dark SaaS template" genérico**: qualquer estética que poderia pertencer a uma startup de fintech, uma plataforma de analytics B2B ou uma ferramenta DevOps. Este produto é inventário agrícola no cerrado brasileiro — deve ser reconhecível como tal.

## Design Principles

1. **Instrumento, não vitrine.** Cor, peso e movimento ganham seu lugar pela utilidade. O verde `#00d68f` é linguagem operacional — sinaliza OK, não decora. Nada existe para impressionar; tudo existe para informar.

2. **Densidade com hierarquia.** Alta densidade de informação é uma feature, não um problema. O objetivo é mostrar mais sem sobrecarregar — via escala tipográfica, agrupamento e contraste, não reduzindo dados. O usuário conhece bem os dados; a interface deve acompanhar esse ritmo.

3. **Enraizado no cerrado.** Este não é um SaaS genérico. Gerencia produtos reais (soja, milho, defensivos, adubos) de uma cooperativa real no Brasil Central. A estética deve carregar essa especificidade — não precisa ser literal, mas deve ser impossível de confundir com uma ferramenta de outro setor.

4. **Precisão calibrada.** Um instrumento profissional tem estética calibrada: espaçamento consistente, escala tipográfica deliberada, cor usada com restrição e propósito. Nada arbitrário. Cada decisão visual deve ter uma razão.

5. **Continuidade operacional.** O usuário retorna diariamente. O design recompensa familiaridade — orientação rápida, padrões previsíveis, zero atrito cognitivo. Calmo sob carga de dados.

## Accessibility & Inclusion

Padrões Material/WCAG AA como linha de base. Contraste alto (texto claro sobre fundo escuro). Sem necessidades especiais declaradas. Tema escuro como padrão permanente — ferramenta de uso interno em ambiente de escritório.
